В рамках повышения безопасности и защиты от атак на этапах авторизации с использованием OAuth 2.0 было принято решение перейти с механизма Code Grant на Proof Key for Code Exchange (PKCE). Ниже приводится информация о выполненных изменениях:

1. **Настройка Keycloak**
    - В конфигурационном файле `realm-export.json` добавлен атрибут `"pkce.code.challenge.method": "S256"`.
    - Выбор метода `S256` обусловлен использованием безопасного хэширования (SHA-256), что делает процесс вычисления `code_challenge` необратимым и значительно повышает уровень безопасности при обмене кодами авторизации.
    - Метод `plain` (передача открытого кода в неизменном виде) не обеспечивал требуемого уровня защиты.

2. **Настройка фронтенда**
    - При использовании библиотеки `@react-keycloak/web` в сочетании с `keycloak-js` была задана опция `initOptions` с параметром `pkceMethod: 'S256'`.
    - Данное изменение гарантирует корректную инициализацию клиента с использованием PKCE и полноценную защиту от атак, связанных с перехватом кода авторизации.

Таким образом, в ходе внедрения PKCE были выполнены все необходимые шаги по настройке и обеспечению дополнительной защиты OAuth 2.0-процессов в приложении BionicPRO.
